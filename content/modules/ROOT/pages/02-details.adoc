== Troubleshooting common setup issues

*Problem*: `oc: command not found`
→ *Solution*: The `oc` CLI is pre-installed in the terminal environment. If you see this error, refresh the browser and open a new terminal session.

*Problem*: `Error from server (Forbidden): namespaces "{user}-database" is forbidden`
→ *Solution*: Your namespace may not have been pre-created. Run `oc new-project {user}-database` to create it manually.

*Problem*: `Login failed (401 Unauthorized)`
→ *Solution*: Verify your credentials: username `{user}` and password `{password}` from the lab interface side panel.

*Problem*: `ImagePullBackOff` when deploying PostgreSQL
→ *Solution*: Run `oc describe pod <pod-name>` and check the Events section. The most common cause is a missing pull secret for `registry.redhat.io`. Ask your facilitator.
