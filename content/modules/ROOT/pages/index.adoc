Welcome to the Deploying databases on OpenShift workshop!

== What you will learn

In this workshop, you will:

* Explain how containerized databases differ from VM-based deployments and when OpenShift is the right choice
* Deploy and configure a production-ready PostgreSQL {postgresql_version} instance with persistent storage
* Connect applications to databases using Kubernetes Services and Secrets
* Apply resource limits, health checks, and credential management for production workloads
* Implement high availability with StatefulSets and Pod Disruption Budgets
* Execute backup and restore using volume snapshots

== Who this is for

This workshop is designed for developers, DBAs, and platform engineers who want to confidently deploy and manage production-grade PostgreSQL workloads on Red Hat OpenShift.

*Experience level*: Beginner to Intermediate

== Prerequisites

Before starting this workshop, you should have:

* Basic Linux command line experience
* Basic understanding of relational databases (SELECT, INSERT, what a primary key is)
* No prior OpenShift or Kubernetes experience required
